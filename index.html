BézierFlow: Learning Bézier Stochastic Interpolant Schedulers for Few-Step Generation

Yunhong Min, Juil Koo, Seungwoo Yoo, Minhyuk Sung
KAIST
Paper arXiv Code

BézierFlow optimizes the sampling trajectory (red) to match the high-quality teacher trajectory (blue) with few steps. By parameterizing the scheduler with Bézier curves, it learns a continuous time-warping function that enables accurate few-step generation.

Abstract

We introduce BézierFlow, a lightweight training approach for few-step generation with pretrained diffusion and flow models. BézierFlow achieves a 2–3× performance improvement for sampling with ≤ 10 NFEs while requiring only 15 minutes of training.

Recent lightweight training approaches have shown promise by learning optimal timesteps, but their scope remains restricted to ODE discretizations. To broaden this scope, we propose learning the optimal transformation of the sampling trajectory by parameterizing stochastic interpolant (SI) schedulers. The main challenge lies in designing a parameterization that satisfies critical desiderata, including boundary conditions, differentiability, and monotonicity of the SNR.

To effectively meet these requirements, we represent scheduler functions as Bézier functions, where control points naturally enforce these properties. This reduces the problem to learning an ordered set of points in the time range, while the interpretation of the points changes from ODE timesteps to Bézier control points. Across a range of pretrained diffusion and flow models, BézierFlow consistently outperforms prior timestep-learning methods, demonstrating the effectiveness of expanding the search space from discrete timesteps to Bézier-based trajectory transformations.

TL;DR

BézierFlow optimizes the sampling trajectory of diffusion/flow models using Bézier curves. It enables high-quality generation in just a few steps (e.g., 4-10 NFEs) by learning a continuous time-warping function that is smooth, monotonic, and boundary-preserving. Training takes only ~15 minutes!

Method

Problem Definition

Our goal is to learn an optimal sampling trajectory that enables high-quality generation with a few Number of Function Evaluations (NFEs), using a pretrained diffusion or flow model. We view this as a teacher-student problem where the teacher follows a high-quality source path (many steps) and the student learns a target path (few steps) that matches the teacher's output.

Formally, we minimize the KL divergence between the teacher's distribution \(q(x_1)\) and the student's distribution \(\bar{p}_\theta(x_1)\):

$$ \min_{\theta} D_{\text{KL}}\left(q(x_1) \Vert \bar{p}_\theta(x_1)\right) $$

In practice, we use a tractable surrogate objective that enforces the outputs of the two solvers to align:

$$ \min_\theta \mathcal{L}(\theta) = \mathbb{E}_{x_0 \sim p_0}\big[ d\big(\xi(x_0, \{t_i\}_{i=1}^N; S_\phi), \bar{\xi}_\theta(x_0, \{s_i\}_{i=1}^M; S_\phi)\big)\big] $$

where \(d(\cdot,\cdot)\) is a distance metric (e.g., LPIPS). This lightweight optimization adjusts only the target scheduler coefficients while using the fixed pretrained model.

Sampling Path Transformation

To define the target path, we employ a scaling reparameterization of the source path. Specifically, we define the target state \(\bar{x}_s\) from the source state \(x_{t_s}\) as:

$$ \bar{x}_s = c_s x_{t_s}, \quad \text{where} \quad c_s = \frac{\bar{\sigma}(s)}{\sigma(t_s)} = \frac{\bar{\alpha}(s)}{\alpha(t_s)} $$

This transformation preserves the endpoint marginals while changing the geometry of the sampling path. The velocity along the target path is given by:

$$ \bar{u}_s(\bar{x}_s) = (\partial_s \log c_s) \bar{x}_s + c_s \frac{dt_s}{ds} u_{t_s}\left(\frac{\bar{x}_s}{c_s}\right) $$

Bézier Parameterization

To learn the optimal scheduler functions \(\bar{\alpha}(s)\) and \(\bar{\sigma}(s)\), we employ 1D Bézier curves. A Bézier parameterization naturally satisfies the critical requirements for a valid scheduler:

  • Boundary Conditions: Fixed control points at \(s=0\) and \(s=1\) ensure the schedule starts at pure noise and ends at data.
  • Monotonicity: Enforcing non-decreasing control points guarantees a strictly increasing Signal-to-Noise Ratio (SNR), ensuring the mapping \(t_s\) is invertible.
  • Differentiability: Bézier curves are smooth (\(C^\infty\)) polynomials, allowing for stable calculation of time derivatives required for the velocity field.
